position: static;

An element with position: static; is not positioned in any special way; it is always positioned according to the normal flow of the page. Static positioned elements are not affected by the top, bottom, left, and right properties:

This div element has position: static;
HTML 
            
                <div class="static">
                This div element has position: static;
                </div>
CSS 
            
                div.static {
                    position: static;
                    border: 3px solid #73AD21; // that's some kind of green
                  }

position: relative;

An element with position: relative; is positioned relative to its normal position, i.e. where it would be with static position. Setting the top, right, bottom, and left properties of a relatively-positioned element will cause it to be adjusted away from its normal position. Other content will not be adjusted to fit into any gap left by the element.

Let's move this green rectangle over 30 pixels by setting the left property to 30px

This div element has position: relative;
HTML 
              
                  <div class="relative">This div element has position: relative;</div>
CSS 
              
            .relative {
               position: relative;
               left: 30px;
               border: 3px solid #73AD21;
            }

position: fixed;

An element with position: fixed; is positioned relative to the viewport, which means it always stays in the same place even if the page is scrolled. The top, right, bottom, and left properties are used to position the element. A fixed element does not leave a gap in the page where it would normally have been located.

Show/Hide fixed element in lower right corner of viewport:

If you scroll down some more, and click the button again, you'll notice that the element remains in the same position within the viewport, no matter where you scroll...

position: absolute;

An element with position: absolute; is positioned relative to the nearest positioned ancestor (instead of positioned relative to the viewport, like fixed). However; if an absolute positioned element has no positioned ancestors, it uses the document body, and moves along with page scrolling.
Note: Absolute positioned elements are removed from the normal flow, and can overlap elements:

This div element has position: relative;
This div element has position: absolute;
HTML 
                
                    <div class="relative-div">
                        This div element has position: relative;
                        <div class="absolute-div">
                          This div element has position: absolute;
                        </div>
                      </div>
CSS 
                
                    .relative-div {
                        margin: auto;
                        position: relative;
                        width: 400px;
                        height: 200px;
                        border: 3px solid #73ad21;
                      }
                
                      .absolute-div {
                        position: absolute;
                        top: 80px;
                        right: 0;
                        width: 200px;
                        height: 100px;
                        border: 3px solid #73ad21;
                      }
